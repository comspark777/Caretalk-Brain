# PlanManager (플랜매니저 페이지)

## 현재 상태
케어톡 위에 **완전 별도 신규 페이지**(`/Admin/PlanManagement/...`)로 구축하는 **플랜매니지먼트(제공기관 인보이스 처리)** 모듈. 우리가 **플랜매니저 + NDIS 케어 서비스 겸업**이라 서비스팀(기존 케어톡)과 물리 분리가 설계 1원칙(COI 차단).
- **파이프라인**: 인보이스 수신(자체 OCR) → 검증 규칙엔진(12체크) → 자동승인/예외 큐/반려 → **퀵클레임 클레임(STEP 4, → [[QuickClaim/_INDEX|QuickClaim]])** → NDIA 입금 대사 → ABA 제공기관 송금 → 송금통지·스테이트먼트·밸런스(Xero 연동).
- **★자금관리 유형 라우팅(STEP 0)**: ①타PM=이메일 발송 ②Agency=`PROVIDER_DIRECT` 클레임(ABA 없음, NDIA가 우리 계좌 직접 입금) ③우리PM=`PLAN_MANAGED` 클레임+ABA 송금 ④자기관리=참가자 직접 인보이스. **②③ 클레임 방식 동일(Bulk Payment Request), `claimType` 플래그만 다름**.
- **원칙**: 자동화 최대화 + 사람 개입 최소화(예외 플래그·은행 송금 실행만 사람). 지급 안전장치=대사 완료 후에만 송금·참가자 자금 신탁계좌 분리·제공기관 은행정보 변경 시 재검증.
- **연동 전제**: OCR=케어톡 자체 내장(Dext 미사용) / Xero=연동 완료(회계·스테이트먼트·지급·대사·ABA) / 퀵클레임=API 계약 확정 예정(공동 개발) / 은행 ABA·NDIS PACE PRODA·이메일SMS=예정.
- **구현 우선순위**: MVP(수신→검증→예외 큐→수동 승인) → 클레임연동 → 대사 → 지급(ABA) → 스테이트먼트/대시보드 → 고도화(Smart Match/Resolve).
